skilla

Agent Skills, installed. Pure bash.

A tiny CLI that installs agentskills.io Agent Skills from any git repo — with dependency resolution, into the .agents/skills/ path agents like Devin index. No Node.js. No runtime. Just bash + git + jq.

⚡ zero Node.js 🧩 resolves dependencies 📁 .agents/skills (Devin-friendly) 🔒 no code runs at install 📜 MIT
View on GitHub Docs 
# install (clone + symlink onto your PATH)
git clone git@github.com:junior/skilla.git && ln -s "$PWD/skilla/skilla" ~/.local/bin/skilla

# then
skilla add git@github.com:acme/skills.git --skill nginx   # installs nginx + its deps

Why skilla

The agent-skills CLI space is crowded — and almost all of it rides on Node/npx. skilla is deliberately small and runtime-free.

No Node.js

One ~800-line bash script. If you have git and jq, you can install skills — on a server, a CI runner, or a locked-down box.

Dependency resolution

A skill's requires: (or plugin.json requiredSkills) are pulled automatically from the same source — transitively.

Project & global scopes

Install into a repo's .agents/skills/ or your home ~/.agents/skills/, tracked in a small JSON registry.

Safe by default

It clones and copies — nothing executes at install time. A skill's scripts only run later, when an agent uses it.

Clean lifecycle

add, update, list, info, remove — registry-scoped, so it only manages what it installed.

Signing-ready

Built to verify signed skills before install (on the roadmap), so a catalog can enforce provenance end-to-end.

bash, not Node

skilla

bash + git + jq. No package manager, no node_modules, no runtime version to match. Copy one file and go.

npx-based tools

Require Node.js + npm and a network fetch of the package each run — awkward on minimal images and restricted environments.

Commands

CommandWhat it does
skilla add <git-url>Install skills from a repo (--skill one, --force reinstall)
skilla listList installed skills + versions
skilla info <skill>Details + declared dependencies
skilla update [skill]Update one, or all
skilla remove <skill>Remove (registry-scoped)
